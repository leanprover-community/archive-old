  1. (you mentioned this one) do we try to unite R-bilinear maps M ‚Üí N ‚Üí P with R-bilinear forms M ‚Üí M ‚Üí R?

  2. do we try to make the theory parallel with R-multilinear maps Œ†(i : Œπ), M‚ÇÅ i ‚Üí M‚ÇÇ?   3. how do we deal with semilinearity, and in particular with the interaction between semilinearity and noncommutativity? Note that currently sesq_form is effectively for a docs#star_ring R, requiring the map be R-linear in one argument and I : R ‚âÉ+* R·µí·µñ-semilinear in the other argument, so a full generalization would need to deal with multiple possible scalar rings (R, R·µí·µñat least, but why not more?).   4. (maybe to be deferred to a future refactor) how does this interact with docs#inner_product_space? I think eventually it would be nice to have something like the integration library, where you can work with a particular measure if you specify it, or with some silent default measure if not -- so here there might in some cases be a silent default bilinear form, and sometimes an explicitly-specified one.
