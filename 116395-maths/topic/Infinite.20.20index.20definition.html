Kevin Buzzard (Oct 06 2021 at 14:21):

I think that subgroup index being 0 when it's infinite is a good idea -- stuff like "order of element divides order of group" works great with this convention. In a parallel universe we wouldn't say "infinite order" for elements of a group, we'd say "order 0" because "g has order n" then means that the ideal of integers ttt such that gt=1g^t=1gt=1 is generated by nnn in all cases. There is no math-nice multiplication on enat because what do we do with 0×∞0\times\infty0×∞? However "infinite index is 0" turns out to be math-nice I think, and well-behaved under multiplication (for example the tower law is true with this convention).

David Wärn (Oct 06 2021 at 19:48):

I agree that with_top pnat would be a natural domain for the index of a subgroup. But this monoid is isomorphic to nat (by identifying ∞\infty∞ with 0), so you may as well work with nat.

One way of understanding this isomorphism is that it maps n : nat to the cardinality of zmod n i.e. the index of nZn\mathbb ZnZ in Z\mathbb ZZ. When you define the order of a group, it's natural to work directly with ideals in Z\mathbb ZZ, and so "order 0" makes a lot of sense. But I don't think it makes any immediate sense to say that the index of a subgroup is 0 (the index is a cardinality, not an ideal) -- it's really only justified by the isomorphism with_top pnat = nat.
