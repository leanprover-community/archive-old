Adam Topaz (Feb 26 2021 at 03:06):

import linear_algebra.tensor_product

open_locale tensor_product

variables {A M N : Type*} [comm_ring A] [add_comm_group M] [add_comm_group N] [module A M] [module A N]

theorem foo {α : M ⊗[A] N} : ∃ L : list (M × N),
  α = (L.map $ λ t : M × N, tensor_product.tmul A t.1 t.2).sum :=
begin
  apply tensor_product.induction_on α,
  { refine ⟨[], by simp⟩ },
  { intros x y,
    refine ⟨[(x,y)], by simp⟩ },
  { rintros x y ⟨L1,hL1⟩ ⟨L2,hL2⟩,
    refine ⟨L1 ++ L2, _⟩,
    simp [hL1, hL2] }
end

Kevin Buzzard (Feb 26 2021 at 09:19):

Here is the reason this came up -- it was in the context of Picard groups of schemes. If SSS is a scheme then we could, perhaps unadvisedly, define the Picard monoid of SSS as the isomorphism classes of sheaves of OX\mathcal{O}_XOX​-modules of SSS under tensor product, and then define the Picard group of SSS as the units in this monoid. This reason this is a bit weird is that we dip our toes into set-theoretic issues here -- the Picard group of SSS is in the same universe as SSS, but the Picard monoid is not (at least for non-empty SSS). Example: if SSS is Spec(K) for K : Type a field then the sheaves are just the KKK-vector spaces, the theory of bases tells us that the Picard monoid is hence the monoid of cardinals (thus in Type 1) but the only solution to X×Y=1X\times Y=1X×Y=1 in cardinals is X=Y=1X=Y=1X=Y=1 so the Picard group is trivial, and hence can be demoted back to Type.

One way of getting around this nonsense is to define the Picard monoid to be the isomorphism classes of sheaves with some finiteness condition (I guess coherent sheaves would be fine in the Noetherian case and perhaps maybe in general -- I know Grothendieck set up some some theory of coherent sheaves on an arbitrary scheme in EGA and that Hartshorne does not use G's definition in his book, he uses a simpler f.g. definition which is equivalent in the Noetherian case but not in general). This more refined definition of the Picard monoid doesn't suffer from any universe issues, so one definitely gets a Picard group which doesn't suffer from set-theoretic issues, but to make sure that one gets the same Picard group one now has to check that every invertible sheaf is coherent.

Here is the proof of this I came up with (of course I could look in the books, but where's the fun in that?): Say $$\mathcal{F}$ is an arbitrary sheaf of OX\mathcal{O}_XOX​-modules, with inverse F−1\mathcal{F}^{-1}F−1. Work over an affine, so everything is really a module over a ring. Write 1∈OX=F⊗F−11\in\mathcal{O}_X=\mathcal{F}\otimes\mathcal{F}^{-1}1∈OX​=F⊗F−1 as ∑ifi⊗gi\sum_i f_i\otimes g_i∑i​fi​⊗gi​, a finite sum, as above. I claim that F\mathcal{F}F is generated by the fif_ifi​. Indeed, for an arbitrary section m∈F=F⊗F−1⊗Fm\in\mathcal{F}=\mathcal{F}\otimes\mathcal{F}^{-1}\otimes\mathcal{F}m∈F=F⊗F−1⊗F we have m=m⊗1=∑im⊗(gi⊗fi)=∑i(m⊗gi)fi=∑irifim=m\otimes1=\sum_i m\otimes (g_i\otimes f_i)=\sum_i (m\otimes g_i)f_i=\sum_i r_if_im=m⊗1=∑i​m⊗(gi​⊗fi​)=∑i​(m⊗gi​)fi​=∑i​ri​fi​ where ri=m⊗gi∈F×F−1=OXr_i=m\otimes g_i\in\mathcal{F}\times\mathcal{F}^{-1}=\mathcal{O}_Xri​=m⊗gi​∈F×F−1=OX​.

What I especially like about this proof is that it looks like the kind of thing you could get away with on the board, especially if you mumble about all the identifications being canonical, but ultimately it relies on the canonical isomorphism M=M⊗RR=M⊗R(N⊗RM)=(M⊗RN)⊗RM=R⊗RM=MM=M\otimes_RR=M\otimes_R(N\otimes_R M)=(M\otimes_R N)\otimes_R M=R\otimes_R M=MM=M⊗R​R=M⊗R​(N⊗R​M)=(M⊗R​N)⊗R​M=R⊗R​M=M being the identity map, an assertion which I am not entirely sure how to convince Lean about :-) I started to formalise the proof in the hope that it might bring some clarity to the situation, but if a passing algebraic geometer wants to bring some clarity too then that would be great :-) Is every invertible sheaf on a scheme coherent?
