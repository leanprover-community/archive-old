Julien Marquet (Mar 27 2022 at 17:59):

I've encountered a similar problem -- I guess it's actually the same situation, but this time it stems from a hidden call to casesOn that is generated by match:

def f (n : Nat) : Option Nat := match n with
  | 0 => some 0
  | n => match f (n - 1) with
    | some m => f m
    | none => none
termination_by f => measure (λ x => match x with
  | some n => n
  | none => 0)
decreasing_by
  trace_state
  admit

The state is :

n✝ n : Nat
⊢ (invImage
        (fun a =>
          measure fun x =>
            match x with
            | some n => n
            | none => 0)
        instWellFoundedRelation).1
    (n - 1) n✝
-- this one is fine

n m : Nat
⊢ (invImage
        (fun a =>
          measure fun x =>
            match x with
            | some n => n
            | none => 0)
        instWellFoundedRelation).1
    m n
-- problem: notice we know nothing about m

(My example is artificial, I made it by isolating the problematic part of a function I wrote in some project of mine that requires a lot more boilerplate to define.)

It is still possible to write the function by breaking it down to a WellFounded.fix call, so this is not a technical limitation.
I just wanted to share this non-trivial nested induction example !

Leonardo de Moura (Mar 27 2022 at 20:02):

@Julien Marquet We usually use subtypes to define this kind of function. The subtype stores the fact that the result is less than equal to the input. 

def f (n : Nat) : Option { r : Nat // r ≤ n } :=
  match n with
  | 0   => some ⟨0, Nat.le_refl _⟩
  | n+1 => match f n with
    | some ⟨m, h₁⟩ =>
      have : m < n+1 := Nat.lt_of_le_of_lt h₁ (Nat.lt_succ_self _)
      match f m with
      | some ⟨r, h₂⟩ => some ⟨r, Nat.le_trans h₂ (Nat.le_trans h₁ (Nat.le_succ _))⟩
      | none => none
    | none        => none

def f' (n : Nat) : Option { r : Nat // r ≤ n } :=
  if h : n = 0 then
    some ⟨0, h ▸ Nat.le_refl _⟩
  else
    match f' (n-1) with
    | some ⟨m, h₁⟩ =>
      have : m < n := Nat.lt_of_le_of_lt h₁ (Nat.pred_lt h)
      match f' m with
      | some ⟨r, h₂⟩ => some ⟨r, Nat.le_trans h₂ (Nat.le_trans h₁ (Nat.pred_le _))⟩
      | none => none
    | none        => none

Leonardo de Moura (Mar 27 2022 at 20:06):

Note that, the following variant 

match n with
  | 0 => ...
  | n => ...

is nasty to write down because we currently do not have a mechanism to tell Lean we want h : n = 0 -> False in the second alternative. This feature is currently only available in the split tactic.
We do have the mechanism for if-then-else as the example above suggests.
