Andrew Yang (Oct 04 2022 at 16:03):

This PR has been sitting for a while now. It introduces the following notation for a presheaf F valued in a concrete category.

  • x ∣_ₕ e for F.map e.op x
  • x ∣_ₗ V ⟪e⟫ for F.map (hom_of_le e).op x
    Some questions (and my thoughts) are as follows

  1. Do we need a notation for this?
    Yes. Some statements are not that readable otherwise.

  2. Are these notations good?
    Debatable. I'd argue this is better that nothing, but this isn't that close what we would write on paper as well.

  3. Will Lean4 enable better notations?
    I have no idea. I don't know much about Lean4.

  4. Should we merge this if there are not better alternatives?
    I'd argue yes.
