Kyle Miller (Jan 24 2022 at 23:12):

For sake of naming consistency, this PR renames tactic.interactive.triv to tactic.interactive.trivial'. The difference between tactic.interactive.trivial and tactic.interactive.triv was that the first uses tactic.triv (and semireducible transparency) and the second uses tactic.triv' (and reducible transparency).

For teaching, @Kevin Buzzard wanted a trivial tactic that really only proved trivial things, but both of these tactics run contradiction if all else fails. I've added tactic.interactive.triv that does the beginning of tactic.interactive.trivial (so no contradiction). It also looks for reflexivity proofs -- does this go too far? should it just be the interactive version of tactic.triv?
