Kyle Miller (Jun 11 2022 at 15:56):

This PR adds instances for different set constructions that result in finite sets. I suppose if s is a set, then the idea is that finite s is the typeclass version of set.finite, and you can use the typeclass system to automatically construct these set.finite proofs (for example, using set.finite_of_finite s).

I found naming things to be somewhat difficult here because there's both the predicate set.finite and the typeclass finite, so results about finite for set can't easily go in set.finite. Does the finite.set namespace seem fine?
