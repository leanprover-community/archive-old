Kyle Miller (Jan 26 2021 at 04:18):

#5813 renames finset.bind to finset.bUnion. It happens to still compile against the most recent master (tested locally) but it touches many files so I worry code rot might set in.

One reason for the change is that finset.bUnion parallels the set API, so it will be easier to find and understand what it's for.

(Maybe another reason to rename it is that there could be a multiset.bUnion defined using a fold of multiset union -- multiset.bind is a fold of multiset sums -- and then finset.bUnion would essentially be equal to multiset.bUnion in that multiset.bUnion (Î» x, (f x).val) s.val = (finset.bUnion f s).val. The equation relating finset.bUnion and multiset.bind is a bit more complicated because it needs a to_finset to remove duplicates.)
