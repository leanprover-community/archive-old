Topic: `injection` introduces too many pis

{% raw %}

Jannis Limperg (Jul 21 2020 at 15:03):

The injection tactic (from core) currently introduces all pis in the target, rather than only the new hypotheses it introduced. Would anyone be unhappy if I changed this? Example:

example (n m k : ℕ) (h : n + 1 = m + 1) : m = k → n = k :=
begin
  injection h with x y z,
  /- actual goal:
     n m k : ℕ
     h : n + 1 = m + 1
     x : n = m
     y : m = k
     ⊢ n = k

     expected goal:
     n m k : ℕ
     h : n + 1 = m + 1
     x : n = m
     ⊢ m = k → n = k

     Nb: we could also clear h:
     n m k : ℕ
     x : n = m
     ⊢ m = k → n = k
  -/
  sorry
end

Related issue: lean#400

Anne Baanen (Jul 21 2020 at 15:13):

I agree that we should change it.

Jannis Limperg (Jul 24 2020 at 12:12):

Seems unanimous, so fixed in lean#407.
