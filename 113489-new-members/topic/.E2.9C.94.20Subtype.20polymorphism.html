Martin Dvořák (Feb 02 2022 at 19:11):

It stops working when I name the subtype.

def foo : ℕ → ℕ := id
def sqrt4 := { x // x * x = 4 }
def two : sqrt4 := ⟨2, by refl⟩
#check foo two

Did I do anything wrong?

Alex J. Best (Feb 02 2022 at 19:13):

When you make a new definition then by default typeclasses (such as has_coe) won't carry over to the new type.
You could make @[reducible] def sqrt4, or define a coercion from nat to sqrt4 in this case.
