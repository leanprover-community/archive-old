Kevin Buzzard (Nov 07 2021 at 22:01):

I would say that this question, whether you do it over the rationals or naturals, is quite tricky to do in Lean for a beginner. You'll need to know how to do algebra under a sum -- the easy rearrangements which humans do can be done by automation like ring -- but rewrites won't work under a binder which makes this one a bit annoying.

Kevin Buzzard (Nov 07 2021 at 22:07):

My (pretty messy) solution

import tactic.ring
import algebra.big_operators.basic
import data.rat.basic

open_locale big_operators
open finset

lemma sum_coe {n : ℕ} : ∑ k in range n, (k : ℚ) = n * (n - 1) / 2 :=
begin
  induction n with d hd,
  { simp },
  { rw [sum_range_succ, hd], push_cast, ring}
end

example {n : ℕ} : ∑ k in range (n+1), (k*(n-k) : ℚ) = (n-1)*n*(n+1)/6 :=
begin
  induction n with d hd,
  { simp },
  { rw [sum_range_succ, nat.succ_eq_add_one,
      nat.cast_add, nat.cast_one],
    -- most is this next line is changing x(d+1-x) to x(d-x)+x using only
    -- axioms, because my usual trick `rw (show x*(d+1-x)=x*(d-x)+x, by ring)`
    -- doesn't work under a binder
    simp_rw [mul_sub, mul_add, mul_one, add_comm, add_sub_assoc,
      ← mul_sub, sum_add_distrib, hd, sum_coe],
    push_cast,
    ring },
end
