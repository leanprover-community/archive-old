Jakob Scholbach (Mar 26 2021 at 09:12):

What does tactic failed, result contains meta-variables mean? Concretely, I have the following (not entirely minimal, I apologize!) example. Near the end (after the 3rd fconstructor) it says "goals accomplished", but at the very end it says tactic failed, result contains meta-variables state: no goals.

import category_theory.category
import category_theory.arrow
import category_theory.limits.preserves.shapes.terminal

open category_theory.limits

namespace category_theory

universes v u v₁
variables {C : Type u} [category.{v} C]

def has_lifting_property (i p : arrow C) : Prop := ∀ (sq : i ⟶ p), arrow.has_lift sq

lemma right_lifting_property_initial_iff (i p : arrow C)
  (h : is_initial i.left) :
  has_lifting_property i p ↔ ∀ {e : i.right ⟶ p.right}, ∃ l : i.right ⟶ p.left, l ≫ p.hom = e :=
begin
  split,
  { sorry },
  { intros h1 sq,
    cases h1 with e he,
    fconstructor,
    fconstructor,
    fconstructor, -- here it says goals accomplished
    },
end -- here it says tactic failed, result contains meta-variables
-- state: no goals

end category_theory

Anne Baanen (Mar 26 2021 at 09:15):

This is usually a bug in a tactic, it basically means there are unsolved goals that were not added to the tactic state. I think you can use the recover tactic to get the goal back.

Scott Morrison (Mar 26 2021 at 10:28):

In any case

lemma right_lifting_property_initial_iff (i p : arrow C)
  (h : is_initial i.left) :
  has_lifting_property i p ↔ ∀ (e : i.right ⟶ p.right), ∃ l : i.right ⟶ p.left, l ≫ p.hom = e :=
begin
  fsplit,
  { intros hlift bottom,
    have comm : (is_initial.to h p.left) ≫ p.hom = i.hom ≫ bottom :=
      is_initial.hom_ext h _ _,
    haveI := hlift (arrow.hom_mk comm),
    use arrow.lift (arrow.hom_mk comm),
    simp },
  { intros h1 sq,
    cases h1 sq.right with e he,
    refine ⟨⟨{lift := e, fac_left' := _, fac_right' := he}⟩⟩,
    apply is_initial.hom_ext, simpa using h, },
end

works.
