One last question about this: If I want to work without tactics, I think I would end up with something along those lines:

lemma comp_part {α β γ : Sort u} (f : α → op β) (g : β → op γ) :
  ∀ x, (comp f g x) = op.no ↔  ( f x = op.no ∨ ( ∃ y : β, f x = op.so y ∧ g y = op.no)) :=
λ x,
  @op.rec
    β
    ( λ (y : op β), f x = y → (comp f g x = op.no ↔ ( f x = op.no ∨ ( ∃ y' : β, f x = op.so y' ∧ g y' = op.no))))
    ( λ (b : β) (k : f x = op.so b), by {unfold comp, rw k, simp,})
    ( λ (k : f x = op.no), by {unfold comp, rw k, simp,})
    (f x)
    rfl

where obviously I cheated in the end, but the outer structure should look like this.
Is there a way to have the same structure using pattern matching? The big problem is: if I match first on f x, I then want to further match on the equality between f x and my just-obtained match for f x, and I don't know if lean supports that. (In agda)
