Hello! I have a (possibly stupid) query:

I was reading Jan Hoffmann's thesis on "Types with Potential", and am trying to code up his linear potential type system from chapter 4 (page 48).

Briefly, the system is:
A::=A ::= A::= unit | bool | int | Lq(A)L^q(A)Lq(A) | Tq(A)T^q(A)Tq(A) | (A,A)(A, A)(A,A)

and he gives some simple rules to compute the potentials of values:

  • Φ(a:A)=0\Phi(a:A) = 0Φ(a:A)=0 if A∈A \in A∈ {unit, bool, int}
  • Φ((a1,a2):(A1,A2))=Φ(a1:A1)+Φ(a2:A2)\Phi((a_1, a_2):(A_1, A_2)) = \Phi(a_1 : A_1) + \Phi(a_2 : A_2)Φ((a1​,a2​):(A1​,A2​))=Φ(a1​:A1​)+Φ(a2​:A2​)
  • Φ(l:Lq(A))=q∣l∣+∑a:lΦ(a:A)\Phi(l : L^q(A)) = q |l| + \sum_{a: l} \Phi(a : A)Φ(l:Lq(A))=q∣l∣+∑a:l​Φ(a:A)
    (and similarly for tree, which I am skipping for brevity)

I coded up the system and function (by using Nat for Int):

inductive HT where
  | Hunit : HT
  | Hbool : HT
  | Hint : HT
  | HList (q : Nat) (A : HT) : HT -- L^q(A)
  | HPair (A : HT) (B : HT) : HT -- (A, B)

def HT.coerce (T : HT) : Type :=
  match T with
  | Hunit => Unit
  | Hbool => Bool
  | Hint => Nat
  | HT.HList _ A => List A.coerce
  | HPair A B => A.coerce × B.coerce

def Φ (A : HT) (a : A.coerce) : Nat :=
  match A with
  | HT.Hunit => 0
  | HT.Hbool => 0
  | HT.Hint => 0
  | HT.HList q A' =>
      List.length a * q + List.foldl Nat.add 0 (List.map (Φ A') a)
  | HT.HPair A₁ A₂ =>
      let (a₁, a₂) := a
      Φ A₁ a₁ + Φ A₂ a₂

Now when I use list or bool, it works fine:

#eval (Φ HT.Hbool false)
-- 0
#eval (Φ (HT.HList 1 HT.Hbool) [false, true, false])
-- 3

But it fails when I try to use nat.

#eval (Φ HT.Hint 0)
-- failed to synthesize instance OfNat (HT.coerce HT.Hint) 0

Is the way I'm coercing the types wrong? Because I can't seem to eval HT.Hint.coerce as it is a Type.
Could somebody help me with this?
