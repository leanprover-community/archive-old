Topic: ✔ Type checking failing on axiom

{% raw %}

Lucas Rufino (May 07 2022 at 17:43):

Hey, I'm new to Lean. I'm using Lean 3.42.1. When I try to create this axiom:

axiom par_assoc (m n l k p q : nat) (d1 : Diag m n) (d2 : Diag l k) (d3 : Diag p q) : (d1 + d2) + d3 = d1 + (d2 + d3)

The console gives me the error:

none of the overloads are applicable
error for Diag.parallel
invalid overload, term
  d1 + (d2 + d3)
has type
  Diag (m + (l + p)) (n + (k + q))
but is expected to have type
  Diag (m + l + p) (n + k + q)

How am I supposed to tell Lean the types are the same? I know I have to use associativity of naturals but i don't know the syntax.

Junyan Xu (May 07 2022 at 17:47):

Haha, this is a classical problem

Lucas Rufino (May 07 2022 at 17:58):

ahhh thanks! I managed to fix it! I changed it to:

axiom par_assoc (m n l k p q : nat) (d1 : Diag m n) (d2 : Diag l k) (d3 : Diag p q) : eq.mpr (by rw [<-nat.add_assoc, <-nat.add_assoc])  ((d1 + d2) + d3) = (d1 + (d2 + d3))

Patrick Johnson (May 08 2022 at 07:15):

Another way would be to use heterogeneous equality:

axiom par_assoc (m n l k p q : ℕ)
  (d1 : Diag m n) (d2 : Diag l k) (d3 : Diag p q) :
  (d1 + d2) + d3 == d1 + (d2 + d3)
